# ROOT/functional/frontmatter

Parses structured metadata from the top of spec node files.

# Public

## Behavior

The frontmatter is an optional YAML block delimited by `---` at
the top of a file. If present, it contains metadata fields that
the framework uses for dependency resolution, artifact tracking,
and external file references.

### Input

A file path.

### Output

A structured record with the following optional fields:

| Field | Type | Description |
|---|---|---|
| `depends_on` | list of strings | Logical names of cross-tree dependencies. |
| `external` | list of externals | References to project files outside the spec tree. |
| `input` | string | Single `ARTIFACT/` logical name for input transformation. |
| `outputs` | list of outputs | Artifacts generated by this node. |

Each **external** entry:

| Field | Required | Description |
|---|---|---|
| `path` | yes | Path to the file, relative to project root. |
| `fragments` | no | List of fragments within the file. |

Each **fragment**:

| Field | Required | Description |
|---|---|---|
| `description` | no | Human-readable description. |
| `lines` | yes | Line range (e.g., `"150-210"`). |
| `hash` | yes | Content hash for staleness detection. |

Each **output**:

| Field | Description |
|---|---|
| `id` | Identifier for the artifact. |
| `path` | File path relative to project root. |

### No frontmatter

If the file has no `---` delimiters, return an empty record
with all fields absent. This is not an error.

## Error conditions

| Condition | Description |
|---|---|
| File unreadable | The file cannot be opened or read. |
| Malformed YAML | The content between `---` delimiters is not valid YAML. |

## Contracts

- The parser reads only the frontmatter block. It never reads
  the file body.
- Unknown YAML fields are silently ignored.
- All recognized fields are optional. An empty frontmatter
  block (`---\n---`) produces an empty record.
